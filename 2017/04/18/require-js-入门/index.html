require.js 入门

发表于 2017-04-18   |  

随着网站功能逐渐丰富，网页中的js也变得越来越复杂和臃肿，原有通过script标签来导入一个个的js文件这种方式已经不能满足现在互联网开发模式，我们需要团队协作、模块复用、单元测试等等一系列复杂的需求。
RequireJS是一个非常小巧的JavaScript模块载入框架，是AMD规范最好的实现者之一。最新版本的RequireJS压缩后只有14K，堪称非常轻量。它还同时可以和其他的框架协同工作，使用RequireJS必将使您的前端代码质量得以提升。

require 能带来的好处

官方对requirejs的描述：
RequireJS is a JavaScript file and module loader. It is optimized for in-browser use, but it can be used in other JavaScript environments, like Rhino and Node. Using a modular script loader like RequireJS will improve the speed and quality of your code.
大致意思：在浏览器中可以作为js文件的模块加载器，也可以用在Node和Rhino环境，balabala…。这段话描述了requirejs的基本功能”模块化加载”，什么是模块化加载

加载不是本地文件

前面提到的是本地文件，如果不是本地文件的情况下，这样子显然是不行的；这就涉及到了require的一个新的内容：–> require.config

以引入一个 jquery 文件来说明

require.config 简单来说就是用来配置模块加载位置的，就是给模块起一个更短更好记的名字，方便我们后期引入；
当然通过paths配置路径还有一个好处就是可以配置多个路径，比如通过CDN加载jquery失败的话，我们可以加载本地下的jquery文件，这就避免了页面中出现加载失败影响整个页面性能的问题的出现；
—需要注意的是，使用require加载模块的时候是不能写后缀名.js 的，否则会报错，因为require已经自动为我们添加了；另外，回调函数中的参数是依赖模块的变量输出，依赖多个模块就可以输出多个变量；需要尽量把有变量输出的写在前面，避免出现错误
